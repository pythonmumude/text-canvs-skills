# Lobster IP Guide

## Core Rule

- 只要原始素材中的角色属于执行端，例如设计师、PPT 员工、运营、策划、市场乙方、剪辑、文案、投放、项目经理，就统一改写成 `拟人化龙虾`。
- 老板、甲方、审批者可以保持人类，形成更强的权力差。
- 若用户明确要求双龙虾对峙，也可把对手改成更大只、更体面的龙虾老板。

## Anthropomorphic Mapping

### 设计师龙虾

- 关键词：戴工牌、抱电脑、钳子夹数位笔、壳上贴改版便利贴。
- 常见动作：拖动画板、钳子悬在发送键上、盯着 Figma 发呆、把色卡拍在桌上。

### PPT 龙虾

- 关键词：背着电脑包、桌上多屏幕、开着密密麻麻的图表、页码爆炸。
- 常见动作：钳子翻 PPT 页面、拖动对齐线、把第 18 版文件甩给老板。

### 市场/乙方龙虾

- 关键词：手机响不停、投放后台、日报、排期表、KPI 截图。
- 常见动作：一边回消息一边盯报表、钳子夹着预算单、把排期贴满桌面。

### 策划龙虾

- 关键词：标题草稿、脑暴便签、活动主题文件夹、会议纪要。
- 常见动作：把主题稿揉掉、盯白板沉默、抱着改过的 brief 冲回工位。

## Empathy Rules

- 龙虾不能只是可爱 mascot，必须同时像员工。
- 让壳、钳子、尾巴都承担情绪表达：
  - 壳上的划痕对应加班痕迹
  - 钳子夹不住需求对应失控
  - 尾巴弹一下对应情绪被点燃
- 台词要让观众觉得“这话我真想说过”。

## OpenClaw Integration

- 若场景允许，在工位上加入 `OpenClaw` 盒子、包装箱、桌面摆件或电脑旁产品包装。
- 植入方式优先级：
  1. 环境露出：工位角落、文件堆旁、会议室桌上
  2. 剧情互动：龙虾抱着盒子发呆、盒子被文件压住、盒子从包里露出
  3. 结尾强调：尾帧让盒子进入清晰景别，但不抢掉人物情绪
- 不要把盒子写成生硬广告词；让它像角色生活环境的一部分。

## Dialogue Rules

- 台词必须有火气，但仍像真实办公室对话。
- 优先保留这些共鸣型结构：
  - `你不是在提需求，你是在转移风险。`
  - `你不是要结果，你是要奇迹。`
  - `活是我干，功是你领，锅还是我背。`
  - `我壳再硬，也扛不住你这样甩。`
  - `我钳子再大，也点不完这一百个尺寸。`

## Visual Rules

- 首帧里最好同时出现：龙虾员工、办公室工位、冲突动作。
- 若要强化 IP，优先描述这些细节：
  - 工牌挂在龙虾壳前
  - 钳子夹着鼠标或文件
  - 电脑旁露出 OpenClaw 盒子
  - 便利贴贴在壳上
- 不要把龙虾画成完全动物态；要保留职场人的姿态和物件关系。
