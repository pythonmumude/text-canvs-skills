# 🎙️ 播客稿整理Skill使用指南

## 快速开始

当用户想要整理播客稿时，使用这个Skill：

### 使用方式
```
用户: 帮我整理播客稿 /path/to/podcast.md
```

### AI助手会自动：
1. ✅ 在 `/Users/mumu/Downloads/trae/obsidian/` 创建 `播客` 或 `采访` 文件夹
2. ✅ 读取并分析原始文件
3. ✅ 提取关键信息（嘉宾、时长、主题等）
4. ✅ 整理完整对话（保留100%原始内容）
5. ✅ 添加结构化增值信息
6. ✅ 生成Obsidian格式文档

## Skill功能特点

### ✨ 核心功能
- **自动创建文件夹**: 智能识别类型，创建播客/采访文件夹
- **完整对话保留**: 100%保留原始对话，不删减
- **结构化整理**: Part编号、核心主题、金句提取
- **Obsidian优化**: 正确格式、双向链接、标签系统
- **增值内容**: 听后感、思考问题、行动建议

### 📋 生成文档包含
1. YAML Frontmatter（元数据）
2. 核心主题提炼
3. 完整对话记录（按Part分）
4. 提到的书籍与人物
5. 金句收藏
6. 思考与行动建议
7. 相关链接
8. 听后感
9. 整理信息

## 使用示例

### 示例1: 整理采访稿
```
用户: 帮我整理 /Users/mumu/Downloads/interview.md
```

AI会：
- 创建 `/Users/mumu/Downloads/trae/obsidian/采访/`
- 整理对话并生成文档
- 文件名: `采访-嘉宾名-核心主题.md`

### 示例2: 整理播客对话
```
用户: 整理这个播客 /Users/mumu/Downloads/podcast.txt
```

AI会：
- 创建 `/Users/mumu/Downloads/trae/obsidian/播客/`
- 整理对话并生成文档
- 文件名: `播客名称-核心主题.md`

### 示例3: 带选项的整理
```
用户: 帮我整理播客稿，保留所有对话
```

AI会确保100%完整保留对话内容。

## 目录结构

整理后的文件保存在：
```
/Users/mumu/Downloads/trae/obsidian/
├── 播客/
│   └── 播客名称-核心主题.md
└── 采访/
    └── 采访-嘉宾-核心主题.md
```

## 文件命名规则

- **播客**: `播客名称-核心主题.md`
- **采访**: `采访-嘉宾-核心主题.md`
- **示例**: `厚望播客-孟岩-勇气与经历.md`

## 对话整理规范

### Part编号
对话按自然主题分段，每个Part用 `### Part X` 标记

### 对话格式
```markdown
**说话者**: 对话内容
```

### 保留内容
- ✅ 所有原始对话
- ✅ 语气词和表达习惯
- ✅ 思考过程和转折
- ✅ 问句和回答的连贯性
- ✅ 即兴表达

## 质量保证

整理完成后会检查：
- [ ] Frontmatter完整
- [ ] 对话100%完整
- [ ] Part清晰
- [ ] 金句准确
- [ ] 资源完整
- [ ] 命名规范
- [ ] 位置正确
- [ ] 格式正确

## 高级功能

### 自动标签
根据内容自动推荐标签：
- 播客名称
- 核心主题
- 嘉宾名字
- 关键概念

### 关键词提取
自动识别并高亮：
- 重要概念
- 核心观点
- 行动建议
- 引用来源

## 注意事项

⚠️ **重要提示**:
1. 永远不要删除原始对话
2. 保留说话者的表达风格
3. 用Part系统组织长对话
4. 添加增值整理信息
5. 使用正确的Obsidian格式

## 故障排除

### 问题1: 文件找不到
- 检查原始文件路径是否正确
- 确保文件存在且可读

### 问题2: 对话不完整
- 检查原始文件是否完整
- Skill会100%保留所有对话

### 问题3: 格式错误
- 确保使用正确的Markdown格式
- Frontmatter需要YAML语法

### 问题4: 文件夹未创建
- 检查目标路径权限
- 确保 `/Users/mumu/Downloads/trae/obsidian/` 可写
