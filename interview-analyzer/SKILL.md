# 采访提问分析器 (Interview Analyzer)

## 核心能力

分析任意采访对话文本，识别和标注以下五个维度：

1. **提问类型** - 开放式/封闭式/假设性/探索式/确认式/对比式
2. **信息层次** - 事实层/行为层/认知层/情感层/价值层
3. **引导技巧** - 垂直追问/水平追问/桥梁转换/对比转换/拉回引导/投射技巧
4. **节奏控制** - 提问密度/停顿时机/语速变化/话题转换节奏
5. **情绪引导** - 安全感营造/情绪激发/情绪调节/情绪共鸣/命名技巧

## 使用方法

当用户提供采访文本时：

1. **读取分析框架**：先阅读 `references/analysis-framework.md` 获取完整的分析维度和标记系统

2. **分析原文**：逐段分析采访者的问题，识别：
   - 每个问题属于什么类型
   - 触及了什么信息层次
   - 使用了什么引导技巧
   - 节奏控制如何
   - 如何处理情绪

3. **创建标注副本**：新建文件（原文件名`_分析版.md`），在原文基础上添加：
   - 每个问题后的行内标注
   - 重要问题的详细批注框
   - 文件末尾的整体结构分析

4. **输出格式**：参考 `assets/annotation-template.md` 的标准格式

## 标注符号系统速查

### 提问类型
- `[O]` 开放式 / `[C]` 封闭式 / `[H]` 假设性
- `[E]` 探索式 / `[CF]` 确认式 / `[CP]` 对比式

### 信息层次
- `[F]` 事实层 / `[B]` 行为层 / `[CO]` 认知层 / `[EM]` 情感层 / `[V]` 价值层

### 引导技巧
- `[VP]` 垂直追问 / `[HP]` 水平追问 / `[RP]` 反问追问
- `[BT]` 桥梁转换 / `[CT]` 对比转换 / `[PT]` 递进转换 / `[RG]` 拉回引导
- `[PR]` 投射技巧 / `[RR]` 角色置换 / `[ES]` 极端情境

### 节奏控制
- `[PD]` 高密度 / `[PM]` 中密度 / `[PL]` 低密度
- `[PSE-N]` 停顿N秒 / `[ACC]` 加速 / `[DEC]` 减速 / `[REP]` 重复关键词

### 情绪引导
- `[EI]` 显性情绪 / `[II]` 隐性情绪
- `[ESF]` 营造安全感 / `[EE]` 激发情绪 / `[ER↑]` 升温 / `[ER↓]` 降温
- `[MIR]` 镜像 / `[VAL]` 验证 / `[LAB]` 命名情绪

## 分析流程

### 步骤1：通读全文
- 识别采访者和受访者
- 标记时间戳（如有）
- 感知整体氛围和节奏

### 步骤2：逐段分析
对每个采访者的问题：

```markdown
**标注格式**：
**[类型 | 层次 | 技巧 | 节奏 | 情绪]** 简要说明

**示例**：
采访者：当时是什么让你决定创业的？
**[O | V | VP | PM | ESF]** 开放式问题触及价值观，垂直追问决策动机，中等密度营造轻松氛围
```

### 步骤3：详细批注（关键问题）
对高价值问题添加完整分析：

```markdown
> **📊 分析批注**
> 
> **提问类型**: [O] 开放式 - "是什么"类型降低防御
> **信息层次**: [V] 价值观层 - 触及决策核心信念
> **引导技巧**: [VP] 垂直追问 + [BT] 桥梁式转换
> **节奏特征**: [PM] 中密度，[PSE-3] 3秒停顿给思考空间
> **情绪引导**: [ESF] 营造安全感，避免"为什么"的质问感
> 
> **💡 学习要点**: 
> - 用"当时"限定时间范围帮助聚焦
> - "是什么"比"为什么"更温和
> - 停顿时机让受访者组织深度回答
```

### 步骤4：整体结构分析
在文件末尾添加：

1. **阶段划分** - 开场/发展/高潮/收尾
2. **提问类型分布** - 统计图表
3. **信息层次递进** - 可视化层次
4. **节奏曲线** - 密度变化图
5. **情绪引导亮点** - 关键时刻分析
6. **话题转换图谱** - 话题跳转逻辑

### 步骤5：可学习清单
- **高价值问题Top 5** - 最值得学习的提问
- **节奏控制大师手法** - 节奏相关亮点
- **情绪引导精彩时刻** - 情绪处理案例
- **可迁移技巧总结** - 通用模板和检查清单

## 输出示例

### 行内标注示例

```markdown
采访者：能聊聊你创业初期的第一个大客户是怎么拿下的吗？

受访者：那时候其实挺偶然的...（回答内容）

**[O | B | VP | PM | ESF]** 开放式问题进入行为层，垂直追问具体案例，中等节奏给予叙述空间，开场不久使用营造安全感
```

### 详细批注示例

```markdown
> **📊 分析批注** (发生在采访第8分钟)
> 
> **提问类型**: [H+O] 假设性+开放式组合
> **信息层次**: [V] 价值观层
> **引导技巧**: [PR] 投射技巧 - 让受访者给"过去的自己"建议
> **节奏特征**: [PD] 高密度前的[PM]缓冲， preceded by [PSE-4] 长停顿
> **情绪引导**: [EE] 激发情绪 + [VAL] 即时验证
> 
> **🎯 技巧解析**:
> 这是整段采访中最精彩的问题之一。通过"给过去的自己建议"这一投射技巧，
> 既避开了直接问"你后悔吗"的尴尬，又触及了深层的价值观反思。
> 
> **💡 学习要点**:
> 1. 投射技巧可以降低防御性，获得更真实的答案
> 2. 长停顿[PSE-4]制造了期待感，让受访者认真思考
> 3. 假设性问题[H]用于测试反思能力，而非获取事实
> 4. 如果受访者卡住了，可以用"哪怕是很小的建议"来降低门槛
```

## 分析原则

### 应该做的 ✅

- 标注每个采访者的问题（受访者回答可选）
- 使用标准符号系统保持一致性
- 提供具体的学习要点和改进建议
- 识别隐性情绪信号（语速变化、用词选择、话题回避）
- 分析话题转换的逻辑和流畅度
- 统计各维度的分布和占比
- 提取可复用的提问模板

### 不应该做的 ❌

- 评判采访者的好坏，而是客观分析技巧
- 过度标注导致原文难以阅读（控制在每2-3个问题有一个详细批注）
- 忽略节奏和情绪维度（这些往往比问题内容更重要）
- 只标注显性技巧而忽略隐性引导
- 遗漏整体结构分析（只给逐段标注是不够的）

## 常见场景处理

### 场景1：多人对话
- 标注每个采访者的提问风格差异
- 分析角色分工（主采访者vs补充提问）
- 识别追问接力（A问B追的组合技）

### 场景2：时间戳缺失
- 根据内容推断段落时长
- 用"早期/中期/后期"代替具体时间
- 重点关注节奏变化而非精确时间

### 场景3：非正式对话
- 识别"非问题式引导"（评论、感叹、沉默）
- 分析闲聊中的信任建立过程
- 标注非语言信号的文字描述（如"（笑）"、"（停顿）"）

### 场景4：对抗性采访
- 重点分析情绪调节[ER↓]技巧
- 识别如何软化尖锐问题
- 分析冲突后的修复策略

## 质量检查清单

分析完成后，确保输出文件包含：

- [ ] 原文完整保留（作为标注基础）
- [ ] 每个采访者问题都有行内标注
- [ ] 至少5个详细批注框（高价值问题）
- [ ] 整体结构分析（阶段划分+统计）
- [ ] 可学习清单（Top问题+技巧总结）
- [ ] 改进建议（哪些地方可以更好）
- [ ] 使用标准符号系统

## 进阶分析提示

当遇到复杂的引导技巧时，参考以下模式：

**组合拳识别**：
- `[VP]→[VP]→[RP]` 连续垂直追问后的反思反问
- `[BT]+[EM]` 桥梁转换触及情感层
- `[ESF]→[EE]→[ER↓]` 安全感→激发→降温的情绪曲线

**节奏陷阱识别**：
- 连续3个以上[VP]会造成疲劳
- [PD]高密度出现在开场是压迫感信号
- [PL]低密度在发展阶段可能失去焦点

**情绪信号识别**：
- 突然出现的[C]封闭式问题可能是情绪防御
- 语速变化描述（"（快速）"、"（缓慢）"）= [II]隐性情绪
- 话题突然跳转可能触碰到[EI]显性情绪边界
